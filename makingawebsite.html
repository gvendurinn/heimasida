Before starting.

To get started I had to start by making a GitHub account. Next we will be installing Brackets, version v2.0.1 in my case. Next step was to install Git, it only takes a few minutes to install both these programs and they are very handy when making a website. Next step was to get the Git extension in Brackets. To do this you go to File>Extension Manager and search for Git Integration.

Be sure that Git Integration is enabled after downloading. Now is a good time to take use of the Getting started feature in Brackets. We can test things out on a .html file already provided with Brackets.

Find a template or make your own.

There are plenty of exciting templates to choose from online so my suggestion is to search for free ones. I do not have the knowledge to start from scratch so I headed on to HTML5 UP! to find an acceptable template. There were many stylish templates to choose from and it took some time to decide. I ended up choosing Solid State.

Setting up the website.

As always when it comes to programming in a new language I started by messing around with the code and using inspect element to guide me. After loads of deleting and finicking around I was pretty happy with the foundation. Most of the work was removing everything I didn't want on the website. The sample html files might have been more useful for a shopping website. It seems like the sample website mostly used <h3> for headings so I continued on using that, also <p> for text and <li> for listed items. Here you can see how to write html as text instead of commands. I didn't do many changes to the sections or the basic structure of the website. When I was in doubt I commented out one line at a time and refreshed the page to see the changes. Here you can see how I changed the items in the Menu by commenting out what I didn't need.

The template was split into three html files for different parts of the site. I started with the index and used that as a template for the rest, it saves time in the long run. After that I made a public repository at Github, then I opened up Git in the website local folder. Easiest way to do that is to right click in the website folder and selecting Git Bash Here. When Git has finished loading I used a code presented when I made the Github Repository, I also had to add my email and username to the config: 

echo "# heimasida" >> README.md
git config --global user.email "you@example.com"
git config --global user.name "Your Name"
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/gvendurinn/heimasida.git
git push -u origin main

To insert code like the one above, use pre and code as shown here: <pre><code>INSERT CODE</code></pre>. Last step is pretty simple, go to Github Pages from your account and selecting the correct branch and folder from the source tab.

You should now be able to see a message saying your website is up and running and a provided link. Interested parties can check out my repository at Github. Feel free to use the html files as templates if you please.

Problems on the way.

It turned out I made a website a few years back that I forgot about and somewhere on the way I accidentally changed the branch in GitHub pages. My page stopped working at that point and it took some time finding the problem. The fix was easy enough, just had to try changing both the branch and folder until it went back up. Probably the most finicky thing about making the website was choosing the correct commands and dimensions for each and every photo. The sample website used both <a href> and <div class> and I still don't have any idea of what the difference is.
